GitHub es una plataforma de alojamiento de repositorios de código que permite a los desarrolladores colaborar en proyectos de software.
Proporciona herramientas para el control de versiones utilizando Git, facilitando la gestión y el seguimiento de los cambios en el código

Los comandos principales de Git que se utilizan con frecuencia en GitHub incluyen:

    • git clone: Descarga un proyecto existente desde un repositorio remoto, creando una copia idéntica del proyecto en tu espacio de trabajo local.
    • git add: Prepara los cambios realizados en archivos específicos o en todos los archivos modificados para ser incluidos en el próximo commit.
    • git init: Inicializa un nuevo repositorio Git en un directorio vacío o en un directorio existente que aún no esté siendo rastreado por Git.
    • git commit: Guarda los cambios preparados en el área de preparación en el directorio git local.
    • git push: Envía los commits al repositorio remoto.
    • git pull: Recibe actualizaciones del repositorio remoto y las aplica en local.
    • git merge: Combina el trabajo realizado en diferentes ramas y es fundamental para el desarrollo colaborativo en Git.
    • git branch: Permite crear, listar y eliminar ramas.
    • git checkout: Cambia entre ramas o permite chequear archivos y commits.
    • git status: Muestra el estado actual del repositorio en tu máquina local, proporcionando información sobre los cambios realizados en los archivos y las ramas actuales.
